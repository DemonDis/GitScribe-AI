## What I do

- Analyze staged changes and generate conventional commit messages
- Suggest appropriate commit types (feat, fix, chore, docs, refactor, test)
- Follow conventional commits specification

## When to use me

Use this when you are about to commit changes.
Run `git diff --cached` or `git status` first to see what's staged.
