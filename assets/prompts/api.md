# Generate API Documentation

Create comprehensive API documentation for the project in Russian.

## Requirements

### Document Structure
1. **API Overview** - brief description of the API's purpose
2. **Authentication** - how to get a token, examples
3. **Base URL** - main endpoint
4. **Endpoints** - all available endpoints with method, path, description, parameters, responses, examples
5. **Data Models** - description of used structures
6. **Error Codes** - standard errors and their descriptions
7. **Limits** - API limitations (if any)

### Style
- Official technical style
- Maximum completeness
- Clear explanations for developers
