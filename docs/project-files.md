# Описание файлов проекта

## Корневые конфигурационные файлы

### `.editorconfig`
Обеспечивает единые настройки редактора (отступы, кодировка, концы строк) для всех IDE — VS Code, WebStorm, Vim и других.

### `.gitattributes`
Управляет тем, как Git обрабатывает файлы. Принудительно устанавливает LF (`\n`) для всех файлов, предотвращая проблемы с концами строк между Windows и Mac/Linux.

### `.gitignore`
Указывает, какие файлы Git должен игнорировать (node_modules, скомпилированный вывод, папки IDE, секреты).

### `.nvmrc`
Говорит `nvm` (Node Version Manager), какую версию Node.js использовать. Автоматически переключает версию при входе в папку проекта.

### `LICENSE`
Лицензия Apache 2.0 — определяет, как другие могут использовать, изменять и распространять код.

### `README.md`
Обзор проекта, возможности, установка, команды и настройка окружения для разработки.

### `CHANGELOG.md`
Записывает все значимые изменения по версиям. Следует формату "Keep a Changelog".

### `CONTRIBUTING.md`
Руководство для контрибьюторов: как сообщать об ошибках, предлагать фичи, отправлять PR и настраивать окружение.

### `CODE_OF_CONDUCT.md`
Кодекс поведения Contributor Covenant. Устанавливает правила общения в сообществе и порядок решения конфликтов.

### `SECURITY.md`
Инструкция по приватному сообщению об уязвимостях через GitHub Security Advisories.

## `.github/` — Настройки GitHub

### `ISSUE_TEMPLATE/bug_report.yml`
Структурированная форма для баг-репортов. Собирает описание, шаги воспроизведения, логи и версию, чтобы issue были полезными.

### `ISSUE_TEMPLATE/config.yml`
Направляет предложения фич и вопросы в GitHub Discussions, оставляя issues только для багов.

### `pull_request_template.md`
Шаблон для pull request. Включает секции: связанный issue, описание, инструкция по тестированию и чеклист.

### `dependabot.yml`
Автоматизирует обновление зависимостей. Раз в неделю проверяет npm-пакеты и создаёт grouped PR, чтобы не засорять уведомлениями.

### `workflows/secret-scanning.yml`
GitHub Actions, который запускает Gitleaks на каждом пуше и PR для обнаружения секретов (ключи API, токены, пароли) в коде.

## `.vscode/` — Настройки VS Code

### `launch.json`
Конфигурация отладки. Нажатие F5 компилирует расширение и открывает новое окно VS Code с загруженным расширением.

### `tasks.json`
Задачи сборки. Определяет шаг компиляции (`npm run compile`) как задачу по умолчанию, которая запускается перед F5 или по Ctrl+Shift+B.

### `settings.json`
Настройки рабочей области: исключает `out/` и `node_modules` из поиска, отключает авто-детект задач tsc.

### `extensions.json`
Рекомендует расширение Extension Test Runner всем, кто открывает проект.

## `.husky/` — Git-хуки

### `pre-commit`
Запускает `npm run compile` перед каждым коммитом. Если компиляция TypeScript не удалась, коммит отменяется.

## Безопасность

### `.gitleaks.toml`
Конфигурация для Gitleaks — сканера секретов. Использует стандартные правила обнаружения ключей API, паролей, токенов и т.д.
